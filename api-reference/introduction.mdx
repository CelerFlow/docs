## What's coming

When the public API ships, you'll be able to:

- Trigger a workflow run programmatically.
- Fetch the status and step output of a specific run.
- List workflows and their recent runs for a team.

Every endpoint will accept a team-scoped API key via `Authorization: Bearer <key>`.

## Generate an API key today

You can already create API keys from the in-product Settings so you're ready the moment the API goes live.

<Steps>
  <Step title="Open API keys">
    From your workspace, go to **Settings → API keys**.
  </Step>
  <Step title="Create a key">
    Name the key so you can identify it later. Team-scoped — it can call any API action the team has access to.
  </Step>
  <Step title="Store it safely">
    The key is shown once. Treat it like a password. If you lose it, revoke it and generate a new one.
  </Step>
</Steps>
