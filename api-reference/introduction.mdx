## Edge Function routing

CelerFlow uses **consolidated Edge Functions** with sub-routes. Each function handles multiple related operations:

| Function | Routes |
|---|---|
| `/agent` | Register, health, terminate, disconnect, remove, policies, confirmations |
| `/fleet` | Create, list, get, delete |
| `/trace` | Ingest spans, query traces |
| `/cli` | Device code, device token |
| `/team` | Create, invite, list members, remove member |
| `/audit` | Create export, list exports, download |
| `/stripe` | Webhook handler |
| `/api-key` | Create, list, revoke |
| `/risk` | Query risk alerts, resolve alerts |
| `/mcp` | Gateway proxy (deployed separately on Railway) |

## Authentication

Most endpoints require a Bearer token:

```
Authorization: Bearer YOUR_TOKEN
```

Two token types:

- **User token** — from the device code OAuth flow. Used for dashboard and fleet operations.
- **Bootstrap token** — assigned to an agent on connection. Used for health reporting, trace ingestion, and agent-specific operations.

## Response format

Success:

```json
{
  "id": "fleet_abc123",
  "name": "Production",
  "created_at": "2026-01-15T10:30:00Z"
}
```

Error:

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Fleet not found",
    "status": 404
  }
}
```

## Rate limits

| Plan | Requests/min |
|---|---|
| Free | 30 |
| Pro | 100 |
| Team | 500 |
| Enterprise | 2,000 |

Rate-limited requests return HTTP 429 with a `Retry-After` header.

## Status codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `201` | Created |
| `400` | Bad request |
| `401` | Unauthorized |
| `403` | Forbidden |
| `404` | Not found |
| `429` | Rate limited |
| `500` | Server error |
