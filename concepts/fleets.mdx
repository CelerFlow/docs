## Overview

A **fleet** is a logical group of agents. Use fleets to organize agents by project, team, environment, or any grouping that makes sense for your organization.

## Why use fleets?

- **Organization** — group related agents together (e.g., "Production", "Staging", "Research team").
- **Access control** — team members see only the fleets assigned to them.
- **Monitoring** — view aggregated health and trace data per fleet.
- **Billing** — usage is tracked per fleet for cost attribution.

## Fleet structure

```
Organization (your Celerflow account)
├── Fleet: Production
│   ├── Agent: code-reviewer
│   ├── Agent: deploy-bot
│   └── Agent: monitoring-agent
├── Fleet: Staging
│   └── Agent: test-runner
└── Fleet: Research
    ├── Agent: data-analyst
    └── Agent: paper-reader
```

## Creating a fleet

<Tabs>
  <Tab title="Dashboard">
    1. Go to the dashboard and click **New fleet**.
    2. Enter a fleet name.
    3. Click **Create**.
  </Tab>

  <Tab title="API">
    ```bash
    curl -X POST https://ristessocgeoiifdbukq.supabase.co/functions/v1/create-fleet \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"name": "Production"}'
    ```
  </Tab>
</Tabs>
