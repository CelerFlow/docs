## Overview

A **trace** is a record of a single tool call made by an agent. Celerflow captures traces automatically for every tool invocation that passes through the policy engine.

## What's captured

Each trace includes:

| Field | Description |
|---|---|
| `timestamp` | When the tool call started (ISO 8601). |
| `completed_at` | When it completed. |
| `agent_id` | The agent that made the call. |
| `agent_name` | Human-readable agent name. |
| `tool_name` | Tool that was called (e.g., `exec`, `gmail`, `write`). |
| `params_summary` | Sanitized parameters (truncated, no secrets). |
| `exec_status` | `success`, `error`, or `blocked`. |
| `policy_result` | `allowed`, `blocked`, or `confirm_required`. |
| `summary` | One-line description of what happened. |
| `duration_ms` | Execution time in milliseconds. |

## Viewing traces

### Dashboard

Navigate to **Traces** in the dashboard to browse, filter, and search traces. You can filter by:

- Agent
- Tool name
- Status (success, error, blocked)
- Time range

### API

Query traces programmatically via the REST API. See the [API reference](/api-reference/introduction) for endpoint details.

## Trace retention

| Plan | Retention |
|---|---|
| Free | 3 days |
| Pro | 30 days |
| Team | 90 days |
| Enterprise | Custom (up to 365 days) |

Traces older than your plan's retention period are automatically deleted by a nightly cleanup job.

Teams on the **Team plan** or higher can export traces as CSV or JSON for compliance, incident investigation, or external analysis.
