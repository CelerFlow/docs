## Overview

Connecting an agent to Celerflow requires two steps: generating a **setup code** from the dashboard, then running `ocauthz connect` from the machine where the agent runs.

## Step-by-step

<Steps>
  <Step title="Generate a setup code">
    In the dashboard, navigate to your fleet and click **Add agent**. A setup code is generated — it's valid for **15 minutes**.

    <Note>
      Each setup code can only be used once. Generate a new code for each agent you want to connect.
    </Note>
  </Step>

  <Step title="Run the connect command">
    On the machine where your agent runs:

    ```bash
    ocauthz connect --code ABC123
    ```

    The CLI:
    1. Sends the setup code to Celerflow for verification.
    2. Registers the agent with a unique ID.
    3. Receives a bootstrap token for future API calls.
    4. Saves the configuration to `~/.ocauthz/agent.json`.
  </Step>

  <Step title="Verify">
    ```bash
    ocauthz status
    ```

    You should see your agent ID, fleet name, and a `connected` status.
  </Step>
</Steps>

## Bootstrap tokens

On successful connection, the agent receives a **bootstrap token**. This token:

- Authenticates all subsequent API calls (health reports, trace uploads).
- Rotates automatically — the CLI handles this transparently.
- Is stored locally at `~/.ocauthz/agent.json`.

<Warning>
  Don't share or commit your `agent.json` file. It contains your bootstrap token.
</Warning>

## Disconnect an agent

To gracefully disconnect:

```bash
ocauthz disconnect
```

The agent is marked as disconnected in the dashboard but its historical data (traces, health checks) is preserved.

## Remove an agent

To permanently remove an agent and all its data:

```bash
ocauthz remove
```

<Warning>
  This action is irreversible. All traces, health checks, and configuration for this agent will be deleted.
</Warning>
