## What is CelerFlow?

CelerFlow is a **governance platform for AI agents**. It gives you visibility into every tool call your agents make, service-level permission controls, and a kill switch when something goes wrong.

One sentence: **"Any agent, one command to connect. See it, control it, kill it."**

<CardGroup cols={2}>
  <Card title="Tracing" icon="timeline">
    Every tool call is recorded and classified — outbound writes are highlighted, local reads are hidden by default. Traces are grouped by task for easy browsing.
  </Card>

  <Card title="Service permissions" icon="shield-check">
    Control access at the service level (Gmail, Notion, GitHub...) with separate read/write toggles. Write operations can require human approval.
  </Card>

  <Card title="Kill switch" icon="power-off">
    Terminate any agent instantly from the dashboard. All tool calls are blocked within seconds via Supabase Realtime push.
  </Card>

  <Card title="Agent score" icon="chart-line">
    Every agent gets a 0–100 health score (A–F grade) based on uptime, success rate, block rate, and response time. Track trends over 7/30 days.
  </Card>
</CardGroup>

## Two ways to connect

CelerFlow supports two integration modes:

| | **OpenClaw plugin** | **MCP Gateway** |
|---|---|---|
| For | OpenClaw agents | Any MCP-compatible framework |
| Setup | `celerflow connect` | Change MCP server URL |
| Coverage | **All tool calls** (built-in + MCP) | MCP tool calls only |
| Latency | Zero (local hook) | ~10–50ms |
| Features | Full (tracing, termination, permissions, HITL, score, token stats) | Tracing, permissions, HITL |

## How it works

<Steps>
  <Step title="Connect your agent">
    Install the CelerFlow plugin or point your MCP client to `gateway.celerflow.ai`. Authenticate with the `celerflow` CLI.
  </Step>

  <Step title="Traces flow automatically">
    Every tool call is recorded with category (outbound write, local read, etc.), service name, parameters, and result. Traces are grouped by task/session.
  </Step>

  <Step title="Set service permissions">
    From the dashboard, configure per-service read/write permissions. Flag dangerous write operations for human approval.
  </Step>

  <Step title="Monitor and act">
    The workspace overview shows agent health scores, risk alerts, token usage, and blocked operations. Hit the kill switch if something goes wrong.
  </Step>
</Steps>
