## Overview

CelerFlow can send notifications when your agents need attention — risk alerts, human approval requests, and status changes. Configure notification channels from **Settings → Notifications** in the dashboard.

## Supported channels

| Channel | Setup | Best for |
|---|---|---|
| **Dashboard** | Always on | All notifications (default) |
| **Telegram** | Bot deep link + QR code | Quick mobile alerts |
| **Slack** | Webhook URL | Team-wide visibility |
| **WhatsApp** | Phone number verification | Critical alerts |

## Setting up Telegram

1. Go to **Settings → Notifications** in the dashboard
2. Click **Connect Telegram**
3. Scan the QR code or click the deep link to open the CelerFlow bot
4. Send `/start` — the bot links to your account automatically

You'll receive notifications for:
- Human approval requests (with approve/deny buttons inline)
- Risk alerts
- Agent status changes (offline, terminated)

## Setting up Slack

1. Go to **Settings → Notifications**
2. Click **Connect Slack**
3. Paste your Slack webhook URL
4. Choose which notification types to forward

## Notification types

| Type | Description | Channels |
|---|---|---|
| **Approval request** | An agent needs human approval for a tool call | All channels |
| **Risk alert** | Anomalous behavior detected | All channels |
| **Agent offline** | Agent missed health checks | Dashboard, Telegram |
| **Agent terminated** | Someone terminated an agent | Dashboard, Telegram |

## Human approval via notifications

When an agent triggers a [human approval](/concepts/human-approval) request, you can approve or deny directly from:

- **Dashboard** — click Approve/Deny in the notification panel
- **Telegram** — tap the inline Approve/Deny buttons
- **Slack** — click the action buttons in the message

Approval requests expire after **5 minutes**. If no response is received, the action is denied by default.
