CelerFlow is in public beta. This page tracks where the product is going. Confidence levels are honest — "Shipping next" is funded and in flight; "Considering" is on the table but not committed.

## Shipping next

- **Public REST API** — programmatically trigger workflows, fetch runs, and list workflows. API keys already exist; the HTTP surface is stabilizing. See [API reference](/api-reference/introduction).
- **Full Settings UI** — team management, API keys, audit log, notifications, and appearance surfaces are being wired into the Settings navigation.
- **Scheduled workflow triggers** — cron-style schedules so workflows can run on their own without a manual or inline trigger.

## Considering

- **More providers** — Google Drive, GitHub, Linear, Google Calendar, and other Composio-backed integrations.
- **Workflow templates marketplace** — pre-built workflows you can clone instead of describing from scratch.
- **MCP gateway** — expose MCP servers and an MCP proxy as a first-class workflow capability.
- **Self-hosted / Docker distribution** — run CelerFlow on your own infrastructure.
- **Team ownership transfer** — move the owner role to another member without recreating the team.

## Not on the roadmap

- **Tool-call-level tracing** — CelerFlow's telemetry is run-scoped and step-scoped, not per-tool-call. See [Runs & executions](/concepts/runs).
- **OpenClaw plugin distribution** — CelerFlow is no longer a plugin for external agent runtimes.
