<Info>
  **Prerequisites**: A CelerFlow account ([celerflow.vercel.app](https://celerflow.vercel.app)) and Node.js v18+.
</Info>

## Connect an OpenClaw agent

<Steps>
  <Step title="Install the CLI">
    ```bash
    npm install -g celerflow
    ```
  </Step>

  <Step title="Log in">
    ```bash
    celerflow login
    ```

    Your browser opens for Google OAuth. After signing in, credentials are saved to `~/.celerflow/credentials.json`.
  </Step>

  <Step title="Create a workspace">
    From the dashboard, click **New workspace**. Give it a name like "Production" or "Dev agents".
  </Step>

  <Step title="Generate a setup code">
    In the workspace view, click **Add agent**. Copy the 6-character setup code (valid for 15 minutes).
  </Step>

  <Step title="Connect">
    ```bash
    celerflow connect --code ABC123
    ```

    The CLI registers the agent, receives a bootstrap token, and saves configuration to `~/.celerflow/agent.json`.
  </Step>

  <Step title="Verify">
    ```bash
    celerflow status
    ```

    You should see your agent ID, workspace name, and `connected` status. In the dashboard, your agent appears with a green health indicator.
  </Step>
</Steps>

## Connect a non-OpenClaw agent (MCP Gateway)

If you're using LangGraph, CrewAI, OpenAI Agents SDK, or any MCP-compatible framework:

<Steps>
  <Step title="Create an API key">
    Go to **Settings → API Keys** in the dashboard and create a key.
  </Step>

  <Step title="Point your agent to the gateway">
    ```bash
    MCP_SERVER_URL=https://gateway.celerflow.ai/mcp
    # Header: Authorization: Bearer cf_key_xxx
    ```

    The gateway federates all your registered MCP servers behind a single endpoint with policy checks and tracing.
  </Step>
</Steps>

## What happens next

Once connected, CelerFlow automatically:

- Records every tool call as a **trace** (categorized by service and operation type)
- Calculates an **agent score** (0–100) updated hourly
- Detects **risk alerts** (token spikes, high block rates, latency anomalies)
- Applies **service permissions** you configure from the dashboard
